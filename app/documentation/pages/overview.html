Wakanda Server Overview

Wakanda Server is a multi-threaded server. The most important components of the server are :

  • JavaScript Application Server
    • Uses V8 as JavaScript runtime
    • Exposes a rich API to handle filesystem, sockets, HTTP Requests, internal or remote databases, workers..
    • Provides a debugger based on Chrome DevTools to debug the different running JavaScript contexts (one context per thread)
  • Wakanda HTTP Server
    • Serves static HTTP files for your applications
    • Used by other server components to expose Wakanda's REST API
    • Can be used by your application's SSJS code to add custom HTTP patterns..
  • Data Abstraction Layer
    • Handles the connection with data-sources (internal WakandaDB or remote databases)
    • Exposes a rich JavaScript API to access and manipualte data
    • Provides events associated to data manipulation
    • Secures access to data
    • Exposes an automatically generated REST API to access and manipulate data
  • WakandaDB (Internal DataBase)
  • Directory
    • Handles users/groups and sessions management as well as authentication
    • Exposes a Server Side JavaScript API
    • Exposes a REST API
