Basics

Whether you are using WakandaDB, an external database or a webservice, we don't use the word Table to describe how a business entity is defined. Instead, we use the word DataClass.

DataClass and Entity

  • A DataClass is a description of a business entity or concept needed in your application.
  • An Entity is an instance of a DataClass.
Instead of manipulating rows you'll be manipulating entities (JavaScript Objects).

Creating your data model


In the Employee DataClass, company is a relation attribute of type Company representing that every employee in the app works for a specific company. The attribute employees in the Company DataClass is a collection of Employee entities.

Once you create and save your model using the model editor, the server automatically generates a REST API to do basic CRUD and even advanced operations (pagination, method calls..).
You can already start the server and switch to the client side section to learn how to access data from the client side.

Attributes

An attribute is an object that stores information or represents a value in the datastore class.
Attributes are by default scalar but you can choose to make an attribute calculated if you need its value to be calculated using code.

Calculated attributes

A calculated attribute is a named property with a data type that masks a calculation.

  • It requires at least the OnGet method to be defined
  • The value of the attribute is not stored
  • If the attribute is not accessed the OnGet method will not be called
  • Inside calculated attribute methods(OnGet, OnSet..) this represents the current entity
  • If you don't define the OnQuery method and use the attribute in a query on large data, expect latency
You can learn more about attributes by visiting the doc-center.

Methods

Datastore class methods allow you to execute code that can be applied to a DataClass, an entity collection, or an individual entity.

  • Inside a collection method this represents the collection who's method was called
  • Inside an entity method this represents the current entity who's method was called

Events

In addition to dataclass methods that are executed when called, Wakanda provides a series of events on both entities and attributes that can trigger specific code to execute. Not to be confused with the On Get, On Set, On Sort, and On Query methods of a calculated attributed, events are available on all attributes and datastore classes.

For both dataclass and attribute events, this points to the entity.

DataClass Events

Class events operate on whole entities and are triggered by particular conditions. The following are datastore class level events:

Event Description
On Init Executes when a new entity comes into being after all auto sequence values are assigned.
On Load Executes just after an entity is accessed from the datastore but before it is delivered to the calling routine.
On Validate Executes when an entity is saved but before the On Save event and can fail the validation if needed.
On Save Executes before an entity is saved and can stop the save if needed.
On Remove Executes before an entity is deleted and can stop the deletion if needed.
On Restricting Query Executes when entities are accessed and returns an entity collection that represents all entities in the dataclass

Attribute Events

Event Description
On Init Executes when a new entity comes into being but after the entity’s On Init event.
On Load Executes the first time the attribute is accessed after the entity is loaded. If the attribute is not accessed, this event is not executed.
On Set Executes after an attribute’s value is set. In server-side code, this executes when the value is set. When an attribute is set in client-side code, this event executes when the entity is saved or the method serverRefresh is called, but only for each attribute that was modified.
On Validate Executes just before the entity’s On Validate event executes and can fail the validation if needed.
On Save Executes just before the entity’s On Save event executes and can stop the save if needed.
On Remove Executes just before the entity’s On Remove event and can stop the deletion if needed.
