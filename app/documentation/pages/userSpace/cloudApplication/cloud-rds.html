To add a RDS group to your application first goto application's group list and click on Add 3rd-party DB button

then configure different options

Finally click on Validate button. At this point your create rds group request are sent, you must wait a few minute to fully provisioned your instance.

General view

The general view gives you real-time used statistics of the memory and disk space usage.

This view also provides the database information for connecting your wakanda application to this database and the Reload Service button for reloading the RDS instance

for how to connect your wakanda application to external database got go doc-center

Monitoring view

The monitoring view allows you to monitor the activity of your RDS instance, by charting the usage statistics of memory, CPU, storage, read/wirte IOPS, Database connections and network throughput.

Graph’s time-frame

You can select a time frame for your graphs by setting a display window as well as a starting date:

Metrics

You can add new metrics to a graph simply by clicking on the + button

then selecting the metric you want from the list, and clicking on Add to graph button

You can also hide or show your selected metrics from the graph by clicking on them.

Delete/add graphs

The x button allows you to delete the graph from the monitoring view.

To add a new graph, click on the Add chart button

the select the metric, period and start date and click on Confirm button

Activity view

This view show you the recent operations made on this RDS instance by you or your collaborators sush as download log, reload, ...

You can refresh list by clicking on refresh button.

profiles view

The profiles view offers you a way to work in a collaborative mode by creating groups, inviting users and managing their permissions.

Creating a group

You can create a new group by clicking on the Add Group button

The Owner group is created automatically with every application. The only user it contains is the application’s owner.

The pop-up that shows up allows you to:

  • Set a name for the group.
  • Type the emails of the collaborators in the Users section.
  • Set custom permissions for the group.

Click on the save button to create the group

Managing users

On the users panel, you can add or remove collaborators from a selected group. To add a new user to a group, click on the Manage Users button or on the Add users to the group link if the group does not contain any users yet.

Type in the user’s email and click on Invite or, if the user is also a collaborator in one of your other groups, select him/her from the shown list.

You can also delete a user from a group by clicking on the X button in the user panel or in the manage users view by deselecting him/her.

Configuring group permissions

You can configure group permissions by clicking on the button.

A predefined list of permissions appears where you can select/deselect the permissions you want to attribute to the group and then save the changes.

Deleting a group

You delete a group by simply clicking on the X button next to the group name.

You should delete all users in a group before deleting it.

Database view

The database view allows you to manage your RDS instance data, from scheduling your backups to exporting your remote data.

Data backups

You can schedule and restore your backups in this view.

To schedule a backup you simply have to set the time you want to start your backups.

You can also find the list of completed backups

Each row contains information about the backup made at that point in time. It includes the date of the backup, its state and the restore action that can be done on it.

Import and export Data

You can easily manage your remote RDS data by importing and exporting your data in this view.

To import your data to your remote RDS instance, you can drag and drop the .sql file in the specified section or select it by clicking on Select File button.

Once the .sql file is selected, click on the Import Data button and your newly imported data will take effect.

To export your remote RDS data, click on the Export Database and the download should start shortly after.

Logs view

The logs view allows you to list all your RDS instance logs.

In addition, You can either download the full logs file in your choice by clicking on in actions column.
